How the loop works

1 · Note it. Add a note here any time — idea, adjustment, bug, question. Tag the module it belongs to.
2 · It reaches Claude. Click Connect sandbox folder once (pick the DMS_ERP_v2_Sandbox folder). Every note then auto-saves to claude_notes.json in the folder.
Notes live by page. The default view groups every open note under its page (01–19). Pick a page in New note and you instantly see everything already queued for it.
Done = cleared. When a note is finished — by you or by Claude — it disappears from the board automatically. That is your signal it has been updated through Claude. Nothing is lost: the Cleared ✓ chip shows the history.
3 · Game plan. In your next Claude session say “check my sandbox notes.” Claude replies to each note and writes a proposed game plan into the same file — it all appears here.
4 · You approve, Claude builds. Press Approve on plan items; Claude makes those adjustments to the sandbox files next session. The live app is never touched.
